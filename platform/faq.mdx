<AccordionGroup>
  <Accordion title="余额充足但提示「额度不足」？">
    **令牌额度**和**账户余额**是分开的。每个令牌有自己的使用上限，即使账户余额充足，令牌额度用完了也会报错。

    解决方法：在控制台编辑令牌，增加额度或开启「无限额度」。
  </Accordion>

  <Accordion title="提示「没有可用渠道」？">
    你请求的模型不在当前令牌的分组内，或该分组的所有渠道都不可用。

    解决方法：换一个包含该模型的分组创建新令牌，或换一个模型试试。
  </Accordion>

  <Accordion title="应该选哪个分组？">
    - 想省钱：`free`（免费）或 `claude-cursor`（0.08）
    - 要大上下文：`claude-ant`（0.15，1M）或 `codex-team`（0.2，1M）
    - 要官方质量：`claude-officially`（1.6）或 `gemini-officially`（1）

    详见[模型与分组](/platform/models-and-groups)。
  </Accordion>

  <Accordion title="请求返回 401 错误？">
    - 检查 API Key 是否正确（注意前后空格）
    - 确认 Key 没有被删除或禁用
    - 确认请求头格式：`Authorization: Bearer sk-你的KEY`
  </Accordion>

  <Accordion title="倍率是什么意思？">
    倍率是价格系数，倍率越低越便宜。例如倍率 0.1 意味着只需官方价格的十分之一。详见[模型与分组](/platform/models-and-groups)。
  </Accordion>

  <Accordion title="支持哪些工具？">
    RuoLi 兼容 OpenAI API 格式，支持所有兼容 OpenAI 的工具，包括：
    - [Claude Code](/tools/claude-code)
    - [Codex](/tools/codex)
    - [Gemini CLI](/tools/gemini-cli)
    - [OpenCode](/tools/opencode)
    - [OpenClaw](/tools/openclaw)
    - [Cherry Studio](/tools/cherry-studio)
    - 以及其他兼容 OpenAI API 的工具
  </Accordion>

  <Accordion title="API 地址是什么？">
    统一地址：`https://ruoli.dev/v1`

    不同工具填写方式略有不同，具体请看各工具的配置页面。
  </Accordion>
