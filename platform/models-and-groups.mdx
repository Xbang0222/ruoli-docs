第一次用这类平台，最容易懵的就是两个词：模型广场和分组。这一页把你当完全没接触过的小白，掰开揉碎讲清楚。

## 模型广场是什么

你可以把 ruoli 想象成一个「AI 模型的大超市」。

市面上做 AI 大模型的公司有很多家——Anthropic 的 Claude、OpenAI 的 GPT、Google 的 Gemini，还有各种国产模型。平时你要用，得分别去每家注册、充值、配置，很折腾。ruoli 把它们都摆到了同一个货架上：你只用一个账号、一个 API Key，就能调用所有这些模型。

这个货架就是[模型广场](https://ruoli.dev/pricing)。打开它，你能看到：

- 现在有哪些模型可以用
- 每个模型多少钱

<Info>
**价格不是固定的。** 就像超市里的菜价会随进货成本浮动，同一个模型在不同时间、不同渠道，价格也会变。所以看价格请以模型广场上显示的为准，文档里不写死具体金额。
</Info>

## 分组是什么

这是最容易让人犯迷糊、但其实特别好懂的一个概念。

还是拿超市打比方。同一样东西（比如一瓶可乐），超市可能从好几个不同的供货商进货：

- A 供货商：便宜，但偶尔缺货
- B 供货商：贵一点，但货源稳定
- C 供货商：还送赠品

AI 模型也一样。同一个模型（比如 Claude），ruoli 背后接了好几条不同的供货渠道——有的便宜、有的稳定、有的「缓存」做得好（重复内容更省钱）。

ruoli 把这些渠道按来源分门别类，每一类就叫一个分组。

你创建 API Key（令牌）时，要选一个分组，相当于挑「走哪条供货渠道」。这个选择决定了三件事：

1. 你能调用哪些模型
2. 价格贵还是便宜
3. 渠道稳不稳定

另外，每个分组都带一个倍率，就是个打折系数——倍率越低越便宜。比如倍率 0.3，差不多就是原价的三折。

<Warning>
**选分组是新手最容易踩的坑。** 如果你选的分组里没有你想用的那个模型，调用时就会直接报错「没有可用渠道」或「模型不存在」。

所以创建令牌之前，先去[模型广场](https://ruoli.dev/pricing)看一眼：你想用的那个模型，在哪个分组里。
</Warning>

## 举两个例子

分组有很多个，这里挑两个最常用的，帮你找找感觉。完整的分组和实时价格，都在[模型广场](https://ruoli.dev/pricing)里：

| 分组 | 倍率 | 什么时候用它 |
|------|------|-------------|
| `cn` | 0.8 | 想用国产大模型 |
| `gpt-pro` | 0.3 | 用 GPT / Codex 写代码，想要稳定 |

## 选错了 / 想换分组怎么办

分组是绑在令牌上的，建好之后不能直接改。想换很简单：重新创建一个令牌、选另一个分组就行，原来的令牌留着或删掉都可以。
